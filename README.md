# A/B-тест карточки объявления: SQL, EDA и статистическая проверка

Учебный портфолио-проект для junior data/product analyst. Данные полностью синтетические и не относятся к реальной компании или продукту. Проект показывает полный аналитический цикл для A/B-теста в сервисе объявлений.

Короткий идентификатор проекта: `ad_card_ab_test_analysis`.

## Бизнес-контекст

В маркетплейсе объявлений команда изменила карточку объявления и кнопку контакта с продавцом. Продуктовая идея: сделать путь к контакту заметнее и увеличить долю пользователей, которые связываются с продавцом после просмотра объявления.

## Цель проекта

Проверить, улучшает ли тестовая версия карточки объявления поведение пользователей по сравнению с контрольной версией.

Основная метрика — `has_contact`: пользователь хотя бы один раз связался с продавцом.

Дополнительные метрики:

- покупки;
- платежи;
- выручка и ARPU;
- число контактов;
- число просмотров.

## Полный аналитический цикл

1. Сгенерированы синтетические данные для продуктового A/B-теста.
2. Создана SQLite-база `data/database/ad_card_ab_test.db`.
3. Написаны и выполнены SQL-запросы для проверки групп, воронки и продуктовых метрик.
4. Проведен EDA: проверка дизайна эксперимента, воронки, метрик и пользовательских распределений.
5. Подготовлен чистый user-level датасет для статистической проверки.
6. Проведены статистические тесты:
   - z-test для двух долей;
   - bootstrap для ARPU;
   - Mann-Whitney U test для revenue.
7. Сформирована финальная продуктовая рекомендация.

## Как запустить проект

Установить зависимости:

```bash
pip install -r requirements.txt
```

Сгенерировать синтетические данные:

```bash
python src/generate_data.py
```

Создать SQLite-базу:

```bash
python src/create_database.py
```

Выполнить SQL-запросы и сохранить витрины:

```bash
python src/run_sql_queries.py
```

После этого можно запускать ноутбуки:

- `notebooks/01_ab_test_eda.ipynb`
- `notebooks/02_ab_test_statistics.ipynb`

## SQL-запросы

- `01_group_sizes.sql` — размеры групп эксперимента.
- `02_event_funnel.sql` — событийная воронка по группам.
- `03_main_metrics.sql` — ключевые продуктовые метрики.
- `04_user_metrics.sql` — пользовательская витрина.
- `05_ab_test_dataset.sql` — чистый датасет для EDA и статистики.

## Текущие результаты

Основная метрика `has_contact`:

- control: 28.60%
- test: 30.60%
- абсолютный uplift: 2.00 п.п.
- относительный uplift: 6.99%
- p-value: 0.016408

Тестовая версия статистически значимо повышает долю пользователей, которые связались с продавцом.

ARPU:

- control: 14.01
- test: 15.09
- bootstrap CI: [-2.18; 4.40]
- bootstrap p-value: 0.527800
- Mann-Whitney p-value для revenue: 0.765537

Статистически значимого эффекта на ARPU не найдено. Выручка разреженная: большинство пользователей ничего не платит, поэтому денежные метрики ожидаемо шумят сильнее контактной конверсии.

## Итоговая рекомендация

Тестовую версию можно рекомендовать к rollout, потому что основная метрика улучшилась статистически значимо. Перед реальным запуском нужно дополнительно проверить технические риски, бизнес-риски, качество контактов и стабильность эффекта по сегментам.

## Ограничения

- Данные синтетические, проект учебный.
- Данные не относятся к реальному сервису объявлений и не описывают реальные пользовательские события.
- Эффект заранее заложен в генератор данных.
- Нет анализа долгосрочного удержания, жалоб, качества сделок и сезонности.
- Вторичные метрики не должны перевешивать основную метрику без отдельной продуктовой причины.
