# Canva slide plan

## Слайд 1. A/B-тест карточки объявления

**Текст на слайде:**

**A/B-тест карточки объявления**
SQL → EDA → статистическая проверка → продуктовая рекомендация

Учебный аналитический проект на синтетических данных сервиса объявлений.

**Визуально:**
Чистый титульный слайд. Можно добавить простую схему: карточка объявления → контакт с продавцом.

**Что сказать:**
Это портфолио-проект, в котором я прошла полный цикл анализа A/B-теста: от подготовки данных и SQL-запросов до статистической проверки и итоговой рекомендации.

---

## Слайд 2. Бизнес-вопрос

**Текст на слайде:**

В продукте изменилась карточка объявления и кнопка контакта с продавцом.

Нужно понять:

- стала ли тестовая версия лучше;
- выросла ли контактная конверсия;
- можно ли рекомендовать rollout.

**Визуально:**

control: старая карточка
test: новая карточка
↓
сравнение метрик

**Что сказать:**
Основная логика такая: если карточка стала удобнее, пользователи должны чаще связываться с продавцом. Поэтому главная метрика — доля пользователей, которые совершили хотя бы один контакт.

---

## Слайд 3. Данные и база

**Текст на слайде:**

Данные синтетические, проект учебный.

В проекте есть 4 таблицы:

- `users`;
- `ads`;
- `events`;
- `payments`.

Итоговая витрина: **12 000 пользователей**.

**Визуально:**

`users` + `ads` + `events` + `payments`
→ SQL
→ user-level dataset

**Что сказать:**
Сначала данные были собраны в SQLite-базу. Потом через SQL я рассчитала размеры групп, воронку событий, основные метрики и подготовила датасет на уровне пользователя для EDA и статистики.

---

## Слайд 4. Проверка эксперимента и EDA

**Текст на слайде:**

Перед тестами я проверила дизайн эксперимента:

- control: 6 000 пользователей;
- test: 6 000 пользователей;
- проверены регион, устройство и сегмент;
- явного перекоса между группами нет.

**Отдельная заметка:**
Выручка разреженная и скошенная — большинство пользователей не платит.

**Визуально:**
График баланса групп или распределения по сегментам.
Можно вставить: `[group balance chart]`.

**Что сказать:**
До статистических тестов важно убедиться, что группы похожи. Если бы группы сильно отличались по регионам или устройствам, результат мог бы быть связан не с изменением карточки, а с перекосом в данных.

## Слайд 5. Основные метрики

**Текст на слайде:**

**Contact conversion**

control: **28.60%**
test: **30.60%**

Абсолютный uplift: **2.00 п.п.**
Относительный uplift: **6.99%**

**ARPU**

control: **14.01**
test: **15.09**

**Визуально:**
Две крупные карточки метрик:

1. Contact conversion: 28.60% → 30.60%
2. ARPU: 14.01 → 15.09

**Что сказать:**
На уровне сырых метрик тестовая группа выглядит лучше. Контактная конверсия выросла на 2 процентных пункта. Но одного сравнения средних недостаточно — нужно проверить, могла ли такая разница появиться случайно.

## Слайд 6. Статистическая проверка

**Текст на слайде:**

Для проверки использованы:

- z-test для contact conversion;
- bootstrap для ARPU;
- Mann-Whitney U test для revenue.

**Результаты:**

| Метрика     | Метод        | Результат         |
| ----------- | ------------ | ----------------- |
| has_contact | z-test       | p-value = 0.0164  |
| ARPU        | bootstrap    | CI: [-2.18; 4.40] |
| revenue     | Mann-Whitney | p-value = 0.7655  |

**Визуально:**
Простая таблица на весь слайд. Значимый результат по `has_contact` можно выделить.

**Что сказать:**
По основной метрике результат статистически значим: p-value меньше 0.05. По ARPU доверительный интервал включает ноль, поэтому уверенно говорить о росте выручки нельзя. Mann-Whitney также не показал значимого различия по revenue.

## Слайд 7. Рекомендация

**Текст на слайде:**

**Рекомендация:*- rollout возможен, но с мониторингом.

Почему:

- contact conversion выросла статистически значимо;
- значимого эффекта по ARPU не найдено;
- перед запуском нужно проверить качество контактов и риски.

**Что дополнительно проверить:**

- жалобы и негативные эффекты;
- стабильность эффекта по сегментам;
- качество контактов;
- долгосрочное влияние на выручку и удержание.

**Визуально:**
Большая карточка решения:

**Rollout: да, но с мониторингом**

Рядом небольшой checklist.

**Что сказать:**
Главная метрика улучшилась, поэтому тестовую версию можно рекомендовать к запуску. Но я бы не делала rollout вслепую: важно убедиться, что контактов стало больше не за счёт ухудшения качества или роста жалоб.

## Слайд 8. Что было реализовано

**Текст на слайде:**

В проекте реализован полный цикл A/B-теста:

- генерация данных;
- SQLite-база;
- SQL-запросы;
- EDA;
- подготовка датасета;
- статистические тесты;
- продуктовая рекомендация.

**Визуально:**

Data → SQL → EDA → Stats → Recommendation

**Что сказать:**
Проект не ограничивается расчётом одной метрики. В нём есть вся цепочка: от данных и SQL до проверки гипотез и итогового решения для продукта.

# Canva design notes

- Формат: 16:9.
- Стиль: простой аналитический шаблон, без лишнего декора.
- Цвета: 1 основной + 1 акцентный.
- Не использовать брендинг Авито и похожие цвета напрямую.
- На слайдах не делать много текста.
- Главные числа вынести крупно.
- Графики использовать только там, где они помогают объяснить вывод.
- Лучше оставить немного пустого места, чем перегрузить слайд.
